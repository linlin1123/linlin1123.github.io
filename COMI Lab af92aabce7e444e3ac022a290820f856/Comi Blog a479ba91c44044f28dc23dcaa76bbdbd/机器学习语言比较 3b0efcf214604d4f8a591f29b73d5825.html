机器学习语言比较

编程语言 —— C++ vs python

1. 适用场景

  • C++
    • 速度快，适用于implement algorithms from scratch ，即实现具体算法
  • python
    • 迭代快，适用于抛弃底层，对machine learning本身的模型进行学习，适用于machine learning user

2. 数据计算库

  • C++
    • Eigen
      • 速度快（相等情况下，numpy跑29s ，Eigen只要0.6s）
      • 文档资料丰富
      • 不依赖其他库
      • 矢量化SSE指令（可结合硬件编程）
    • Armadillo
      • 矩阵类型以及矩阵函数丰富
      • 封装好，对用户更友好
      • 最终速度，由Armadillo+其他数据库的组合而定
      • Armadillo+OpenBLAS 的组合比Eigen快，可优化了inter CPU计算
  • python
    • numpy
      • 操作矩阵运算很方便，对用户友好
      • 速度不及C++的计算库

3. 机器学习库

  • C++
    • mlpack
      • http://www.mlpack.org/
    • shark ML library
      • http://image.diku.dk/shark/
    • dlib
      • http://dlib.net/
    • 特点
      • 提供机器学习的基本算法：K-means，SVM，多层perceptron，PCA，deep learning等，详情可见官网docs
      • 三种库褒贬不一，根据具体情况进行选择
  • python
    • scikit-learn
      • 拥有大量的数据挖掘和数据分析功能
      • 支持的机器学习算法相较于C++的库更强大
    • Theano
      • 支持CPU / GPU运算
      • 只支持python接口
      • 整个代码库都是python的，难以调试、重构和维护
      • 支持大部分先进的网络，符号API支持循环控制，让RNN的实现更加容易简洁

4. 其他比较

  • 效率
    • C++效率高，Tensorflow以及Torch的底层实现都是C++
  • 适用的机器学习方向
    • C++
      • deep learning
      • computer vision
      • 序列分析
      • 手势识别
      • 通用机器学习
      • 自然语言处理
    • python
      • data analytics（譬如预测走势或推荐算法）
      • data mining
      • 通用机器学习
      • 自然语言处理
    • 各语言侧重性目录
      • https://yq.aliyun.com/articles/43089?utm_content=m_29956
  • 支持
    • python库较多，且社区支持更好
  • 数据处理
    • 在数据获取、处理、展示等方面，python均强于C++

5. 总结

  • 选择参考
    • 若是针对Machine Learning本身 的使用，应选择python
      • 机器学习算法方面有很多现成库
      • 数据处理方便
      • 编程学习成本低
    • 若是针对要实现或改进机器学习具体算法 的情况，应选择C++
      • 效率高
      • 更好地了解算法的实现，有助于机器学习模型参数的选择，以及模型的生成
  • 折中做法
    • python进行prototype
      • 利用python编码简单易于迭代且库多的优势，先在平台上生成可用的机器学习模型原型
    • C++实现产品
      • 借助C++效率高的优势，将在python中选择的模型，用C++进行编程实现，形成最终的产品
